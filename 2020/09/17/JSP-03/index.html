JSP 03 9 jsp implicit objects

Posted on 2020-09-17 | In JSP

9 jsp implicit objects

  • after tomcat translation, it will automatically create 9 obj
    including
  1. request
  2. response
  3. pageContext
  4. session
  5. application (ServletContext obj)
  6. config (ServletConfig obj)
  7. out
  8. page
  9. exception

4 domian obj

  • domain obj is used to save and read data
  • the usage of 4 domain obj is the same
  • only the scope is different
  1. pageContext (current jsp page)
  2. request (current request)
  3. session (current session, until browser is close)
  4. application (current web module, until it stop)

Difference between out and response.getwriter()

  • response will output first
  • because both out and response will write to its own buffer
  • when writing to output, the programme will first execute out.flush()
  • which will flush all the element inside out’s buffer to the response’s buffer
  • and then output all element in response’s buffer
  • since the jsp use out for output
  • so out is recommend

out.write() and out.print()

  • write() can only output string
  • print() will change other class to string and then call write()
  • recommend use print()
