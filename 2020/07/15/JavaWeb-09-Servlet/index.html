JavaWeb 09 Servlet

Posted on 2020-07-15 | In Server , Servlet

Servlet

What is Servlet

  • Servler is a java interface
  • It is a java programme
  • which is used to receive the request from client and response data back to client

implement servlet

  • workflow
  1. write a class to implement servlet / extends HttpServlet Class(most case)
  2. implement service() method
  3. configure web.xml for the servlet to match a address

web.xml configuration

  1. configure the <servlet> and <servlet-mapping> tag

life cycle of servlet

  1. execute Servlet constructor
  2. execute init() method
  • first two step will be executed when the url is first access and only execute once
  1. execute serivice() method (service method will execute on every access)
  2. execute destroy() method (when the web project stop)

Handle GET and POST separately

  • use HttpServletRequest to separate get and post

HttpServlet

What is HttpServlet?

  • HttpServlet is a class which implemented Servlet interface
  • In most cases, programmer only need to inherit the HttpServlet class instead of Servlet

Implement HttpServlet

  • Workflow
  1. implement a class to inherit HttpServlet
  2. rewrite doPost() and doGet accroding to depending on your needs
  3. configure the web.xml (same as above)

Servlet Inheritance Structure

SevletConfig

  • both servlet and servletconfig is created by tomcat

  • Usage of SevletConfig

  • get the <servlet-name> <init-param> <servlet-context> in the web.xml

Rewrite init()

  • if init() is overrided, super.init(config) must be call when rewriting init() method

ServletContext

What is ServletContext?

  • ServletContext is a interface
  • It is a domain object, a web project only contain one ServletContext obj
  • ServletContext obj is created when the proj. is initial and destroy when proj. stop

method

  • setAttribute()
  • getAttribute()
  • removeAttribute()
  • getContextPath()
  • getRealPath()

Usage

  • get the <context-para> in the web.xml
  1. get the current project path
  2. get the actual harddisk path in the server
  3. CRUD data
