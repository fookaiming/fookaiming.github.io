JSP 03 9 jsp implicit objects

Posted on 2020-09-17 | In JSP

9 jsp implicit objects

  • after tomcat translation, it will automatically create 9 obj
    including
  1. request
  2. response
  3. pageContext
  4. session
  5. application (ServletContext obj)
  6. config (ServletConfig obj)
  7. out
  8. page
  9. exception

4 domian obj

  • domain obj is used to save and read data
  • the usage of 4 domain obj is the same
  • only the scope is different
  1. pageContext (current jsp page)
  2. request (current request)
  3. session (current session, until browser is close)
  4. application (current web module, until it stop)

Difference between out and response.getwriter()

  • response will output first
  • because both out and response will write to its own buffer
  • when writing to output, the programme will first execute out.flush()
  • which will flush all the element inside out’s buffer to the response’s buffer
  • and then output all element in response’s buffer
  • since the jsp use out for output
  • so out is recommend

out.write() and out.print()

  • write() can only output string
  • print() will change other class to string and then call write()
  • recommend use print()

JSP 01

Posted on 2020-09-17 | In JSP

What is JSP

  • JSP Programme(replace servlet) to send html page to browser
  • Because using servlet to send html page is clumsy
  • below example shows using servlet to send html page

access Jsp page

  • jsp store under web directory
  • access jsp page is same as html page
  • JSP is actually a servlet programme

translation

  • when accessing jsp page at first time, tomcat will translate the jsp page to a .java file and .class file.
  • by looking at the .java file we can see it is a class extends HttpSevlet
  • the translated java class is more or less the same as the exp. above (using writer to send html page to browser)

JSP header

attributes in the header

  • language (only java is allowed)
  • contentType
  • pageEncoding
  • import (import java class)
  • autoFlush (for the writer in jsp)
  • buffer (for the writer in jsp)
  • errorPage (when error occurs forward to erroPage)
  • isErrorPage
  • session (true if need to create a HttpSession obj)
  • extends (what class need to be extends for the jsp)

JavaWeb 13 Ajax

Posted on 2020-08-30 | In Server , Servlet

HttpServletRequest

  • whenever a request access the tomcat server, it will parse the request to a Request obj and pass it to the service() method(doGet/ doPost), we can cast it to HttpRequest to get the infos about the request

Common API

  • getRequestURI()
  • getRequestURL()
  • getRemoteHost()
  • getHeader()
  • getParameter()
  • getParameterValues()
  • getMethod()
  • setAttribute(key, value)
  • getAttribute(key, value)
  • getRequestDispatcher

Encoding

  • post request may causes encoding problem
  • use req.setCharacterEncoding(“UFT-8”) to solve the problem
  • it must be use before accessing any parameters

Redirect

HttpServletResponse

  • whenever a request access the tomcat server, it will parse the request to a Response obj and pass it to the service() method(doGet/ doPost), we can cast it to HttpRequest to get the infos about the request

Common API

  • getOutputStream()

  • getWriter()

  • the above two method can either choose one to use

Encoding

  • use resp.setCharacterEncoding(“UFT-8”) to set the server encoding
  • and use resp.setHeader(“Content-Type”,”text/html; charset=UTF-8”) to set the browser encoding
    or
  • use resp.setContentType(“Content-Type”,”text/html; charset=UTF-8”) to set both server and browser encoding
  • this method must be use before getWriter()

Redirect
