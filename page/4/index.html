JSP 01

Posted on 2020-09-17 | In JSP

What is JSP

  • JSP Programme(replace servlet) to send html page to browser
  • Because using servlet to send html page is clumsy
  • below example shows using servlet to send html page

access Jsp page

  • jsp store under web directory
  • access jsp page is same as html page
  • JSP is actually a servlet programme

translation

  • when accessing jsp page at first time, tomcat will translate the jsp page to a .java file and .class file.
  • by looking at the .java file we can see it is a class extends HttpSevlet
  • the translated java class is more or less the same as the exp. above (using writer to send html page to browser)

JSP header

attributes in the header

  • language (only java is allowed)
  • contentType
  • pageEncoding
  • import (import java class)
  • autoFlush (for the writer in jsp)
  • buffer (for the writer in jsp)
  • errorPage (when error occurs forward to erroPage)
  • isErrorPage
  • session (true if need to create a HttpSession obj)
  • extends (what class need to be extends for the jsp)

JavaWeb 13 Ajax

HttpServletRequest

  • whenever a request access the tomcat server, it will parse the request to a Request obj and pass it to the service() method(doGet/ doPost), we can cast it to HttpRequest to get the infos about the request

Common API

  • getRequestURI()
  • getRequestURL()
  • getRemoteHost()
  • getHeader()
  • getParameter()
  • getParameterValues()
  • getMethod()
  • setAttribute(key, value)
  • getAttribute(key, value)
  • getRequestDispatcher

Encoding

  • post request may causes encoding problem
  • use req.setCharacterEncoding(“UFT-8”) to solve the problem
  • it must be use before accessing any parameters

Redirect

HttpServletResponse

  • whenever a request access the tomcat server, it will parse the request to a Response obj and pass it to the service() method(doGet/ doPost), we can cast it to HttpRequest to get the infos about the request

Common API

  • getOutputStream()

  • getWriter()

  • the above two method can either choose one to use

Encoding

  • use resp.setCharacterEncoding(“UFT-8”) to set the server encoding
  • and use resp.setHeader(“Content-Type”,”text/html; charset=UTF-8”) to set the browser encoding
    or
  • use resp.setContentType(“Content-Type”,”text/html; charset=UTF-8”) to set both server and browser encoding
  • this method must be use before getWriter()

Filter

What is Filter

  • It is a interface under JavaEE
  • it can be used to intercept requests, filter response
  • common useage: Permission check, log, transcation management

Life cycle

  • contructor and init() start when web project starts
  • doFilter() exe when any request matches the url pattern
  • destroy() exe when web project ends

FilterConfig

  • FilterConfig obj can used to get filter-name, init-param and servletContext

    How to implement Filer

  • write a class implement Filter
  • impl doFilter()
  • config filter tag in web.xml

Example

  • the prority of different filter depends on the writting position in web.xml
  • multiplt filters use the same thread and request obj in default

JavaWeb 09 Servlet

Posted on 2020-07-15 | In Server , Servlet

Servlet

What is Servlet

  • Servler is a java interface
  • It is a java programme
  • which is used to receive the request from client and response data back to client

implement servlet

  • workflow
  1. write a class to implement servlet / extends HttpServlet Class(most case)
  2. implement service() method
  3. configure web.xml for the servlet to match a address

web.xml configuration

  1. configure the <servlet> and <servlet-mapping> tag

life cycle of servlet

  1. execute Servlet constructor
  2. execute init() method
  • first two step will be executed when the url is first access and only execute once
  1. execute serivice() method (service method will execute on every access)
  2. execute destroy() method (when the web project stop)

Handle GET and POST separately

  • use HttpServletRequest to separate get and post

HttpServlet

What is HttpServlet?

  • HttpServlet is a class which implemented Servlet interface
  • In most cases, programmer only need to inherit the HttpServlet class instead of Servlet

Implement HttpServlet

  • Workflow
  1. implement a class to inherit HttpServlet
  2. rewrite doPost() and doGet accroding to depending on your needs
  3. configure the web.xml (same as above)

Servlet Inheritance Structure

SevletConfig

  • both servlet and servletconfig is created by tomcat

  • Usage of SevletConfig

  • get the <servlet-name> <init-param> <servlet-context> in the web.xml

Rewrite init()

  • if init() is overrided, super.init(config) must be call when rewriting init() method

ServletContext

What is ServletContext?

  • ServletContext is a interface
  • It is a domain object, a web project only contain one ServletContext obj
  • ServletContext obj is created when the proj. is initial and destroy when proj. stop

method

  • setAttribute()
  • getAttribute()
  • removeAttribute()
  • getContextPath()
  • getRealPath()

Usage

  • get the <context-para> in the web.xml
  1. get the current project path
  2. get the actual harddisk path in the server
  3. CRUD data

JavaWeb 07 jQuery II

Posted on 2020-07-13 | In JavaWeb , JavaScript

jQuery

Diff. between window.onload and $(function)

  • $(function) will execute immediately after the dom obj is ready
  • window.onload will execute after the dom is ready and all content is loaded. (eg. img, iframe tag)
  • when multiple window.onload is used, only the last one will execute
  • all $(function) will execute one by one

Event

  • activate when not passing para.
  • register when passing para.
    click()
    mouseover()
    mouseout()
    bind() register multiple events one() same as bind(), but the events only response once
    unbind()
    live() same as bind(), but the events also work for the elements after binding. see exp. below

Event Bubble

  • when two element is nested and register the same event, when the event is activate by the child element, it will also propagate to the parent element.

How to prevent bubbling

  • return false in the child function

Event Obj

  • Event obj will automatically create when an event is activated.

  • a lot of information about the event is store in the event obj.

  • How to access the event obj

  • use “event” as a para. in the event function

  • common use of event obj

  • when bind is used to bind mutilple evnets, event can be used to find out the activated event
