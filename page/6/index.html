  • child class will automatically inheritance the attributes and methods from it parents - private methods cannot be inheritated

  • cannot lower the method’s access modifer

Overload(same method name with diff parameter)

  • overload method should always have diff parameter, cannot change the return type only

Wildcard Filtering

LIKE

  • use LIKE with wildcard (%, _ ) to create a search pattern to match record
  • if not wildcard is used with LIKE, LIKE act as equal operator

Wildcard

  • dont use wildcard in the begining of the search if not necessary -> low efficiency
    ### % operator
  • match any number (including 0) of occurrences of any character
  • can be occurrence in any place in the wildcard
  • cannot use to match null value

_ operator

  • similar as % but only match with one character
  • cannot match 0 occurence

Sorting

ORDER BY

  • it can apply to the selected column and the non selected column. see exp. below

  • apply to mutiple column (sort by the first column, if the same, sort by the next columns)

DESC (reverse order, descending)

  • DESC is only apply to its precedence column

ASC (ascending)

  • default sorting
  • not vert useful

Tips

  • find maximum or minimum by ORDER BY and LIMIT
  • Position of ORDER BY Clause When specifying an ORDER BY clause, be sure that it is after the FROM clause. If LIMIT is used, it must come after ORDER BY. Using clauses out of order will generate an error message.
